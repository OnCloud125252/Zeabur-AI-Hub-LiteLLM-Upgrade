## 摘要

此 PR 改良了最初在 PR #16895 中引入的 Gemini thought signature 功能。它將此功能從 beta 狀態移除，並透過用於檢查 tool calls thought signatures 的 pre-call hook 改進了實作方式。

### 主要改進

1. **移除 beta 狀態** - thought signature 功能不再被視為實驗性質
2. **新增 pre-call hook** - 專用的 hook 在發送請求前檢查 tool calls 的 thought signatures
3. **增強相容性** - 改進對 OpenAI Agents SDK 的支援

## 從 Beta 遷移

先前使用 beta 功能的使用者請注意：

1. `__thought__` 分隔符格式保持不變
2. Tool call ID 繼續包含以下格式的 signatures：`call_<uuid>__thought__<signature>`
3. 此功能現在無需 beta 設定即可自動套用
4. Pre-call hook 確保更好的錯誤處理和驗證

## 使用範例

Tool call 流程可無縫運作：

1. **第一個請求**：客戶端發送 tool call 請求
2. **Gemini 回應**：回傳 tool call，signature 嵌入在 ID 中

   ```json
   {
     "tool_calls": [{
       "id": "call_xxx__thought__CsoDAePx...",
       "function": { "name": "calculator", "arguments": "{...}" }
     }]
   }
   ```

3. **後續請求**：客戶端回傳具有相同 tool call ID 的 tool 結果
4. **Pre-call Hook**：LiteLLM 從 ID 中擷取 thought signature
5. **Gemini API**：接收適當的 signature 以維持上下文連續性

## 影響

此 PR 將 thought signature 功能鞏固為 LiteLLM Gemini 整合中穩定且可用於正式環境的元件。這些改進實現了：

- 更可靠的 Gemini 3 模型 tool calling
- 更好的 agent 框架相容性
- 使用 pre-call hook 模式的更簡潔程式碼架構
- 降低因缺少 thought signatures 而導致 400 錯誤的風險
